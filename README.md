# AFAD: 計算・モデル二重異種性に対応する適応的連合アーキテクチャ分散学習

**DICOMO2026 投稿論文**
著者: 島野 凌，須山 敬之（大阪工業大学 大学院 情報科学研究科）

---

## 論文概要

連合学習（FL）の実環境では，端末ごとの計算能力の差（計算異種性）と，
CNNやTransformerなどアーキテクチャの違い（アーキテクチャ異種性）が同時に存在する．
既存手法のHeteroFLは計算異種性のみ，FedGenはアーキテクチャ異種性のみに対応しており，
両者を同時に解決する手法は存在しなかった．

本研究では，**32次元共有ボトルネック層**を接点としてHeteroFLとFedGenを統合した
フレームワーク **AFAD（Adaptive Federated Architecture Distribution）** を提案する．

OrganAMNISTを用いたIID環境での実験により，AFADは**88.89%** の精度を達成し，
FedGen（86.32%）を上回るとともに，HeteroFLと同等以上の精度を示した．
これはIID環境において，二重異種性を同時に扱うことが精度トレードオフを生じないことを示す．

##### 調査結果

**問題1: Generator 損失**
- `afad_generator_trainer.py` の実装：`CE(f_a(G(z,y)), y)` + `diversity_loss`
- 実装は**クロスエントロピー最小化**（Generator がクラス y を正しく予測できる latent を生成する）
- 論文の「エントロピー最大化」説明・式 $\sum_k p_k \log p_k$ は**完全に誤り**（実装と逆の目標）

**問題2: $\mathcal{L}_{KD}^{ens}$ の教師信号**
- `afad_client.py` の実装：2種の損失はどちらも Generator が唯一の教師
  - Term1 (L_KD^gen)：`CE(client_model(G(y_rand)), y_rand)` — ランダムラベルで識別知識を転写
  - Term2 (L_KD^ens)：`KL(client_model(x_real) || client_model(G(y_real)))` — 実データとGeneratorサンプル間の一致
- 「全端末のアンサンブル出力」は使われていない；Generator がアンサンブルの代替として機能
